FALAFINA v0.7 — COMO COLOCAR NO RENDER (passo a passo)
========================================================

O QUE MUDOU NESTA VERSÃO
- Contas agora são salvas NA NUVEM (servidor) — nunca mais se perdem.
- Login funciona de QUALQUER aparelho.
- Ranking agora é MUNDIAL (todos os alunos).
- Teste de nivelamento: quem já sabe inglês não começa do zero.
- Atalho pro Simulador de Entrevista pra alunos intermediários/avançados.
- Ícones corrigidos (antes davam 404 e quebravam a instalação do app).
- Sincronização offline-first: sem internet tudo funciona; quando
  conecta, o progresso sobe sozinho. O servidor NUNCA apaga XP: só soma.

ESTRUTURA
  server.js        -> o servidor (Node puro, zero dependências)
  package.json     -> diz ao Render como iniciar (npm start)
  publico/         -> o site (index.html, sw.js, manifest, ícones)

PASSO A PASSO NO RENDER
1. Suba esta pasta num repositório do GitHub (pode ser novo: "falafina").
2. No Render: New + -> Web Service -> conecte o repositório.
3. Configure:
   - Runtime: Node
   - Build Command: (vazio ou "npm install")
   - Start Command: npm start
4. IMPORTANTE — O DISCO (sem isso os dados somem a cada deploy!):
   Aba "Disks" do serviço -> Add Disk:
   - Name: dados
   - Mount Path: /data
   - Size: 1 GB (dá pra dezenas de milhares de contas)
   Obs: Disk exige plano pago (Starter, ~US$7/mês). No plano Free
   NÃO existe disco persistente — os dados seriam apagados a cada
   deploy/restart. Pra "salvar pra sempre", o disco é obrigatório.
5. Deploy. O site e a API ficam no MESMO endereço
   (ex.: https://falafina.onrender.com).
6. Teste: abra https://SEU-SITE.onrender.com/api/saude
   Deve responder: {"ok":true,...,"discoPersistente":true}

MIGRANDO DO NETLIFY
- O Netlify só serve arquivos — não roda o server.js. A partir da v0.7,
  o site oficial deve ser o endereço do Render.
- Se quiser manter o mesmo domínio, aponte-o pro Render (Settings ->
  Custom Domains). Vantagem: o localStorage antigo dos usuários continua
  valendo e a primeira sincronização sobe tudo pra nuvem sozinha.
- Se o endereço mudar, os poucos usuários atuais precisam criar a conta
  de novo no site novo (o progresso antigo fica no aparelho deles no
  endereço antigo).

BACKUPS
- 1 backup por dia em /data/backups (guarda os últimos 20).
- Se o arquivo principal corromper, o servidor restaura sozinho do
  backup mais novo ao reiniciar.
